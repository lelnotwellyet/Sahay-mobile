Fig 3.9 — Sequence Diagram (Video Consultation)

sequenceDiagram
    actor C as Client App
    participant SDB as Supabase DB
    participant SRT as Supabase Realtime
    actor P as Psychiatrist App
    participant STUN as Google STUN

    Note over C,P: Session Request Phase
    C->>SDB: INSERT session (status=pending)
    SDB-->>C: session_id

    loop Poll every 3 seconds
        C->>SDB: SELECT status WHERE id=session_id
        SDB-->>C: status=pending
    end

    P->>SDB: SELECT pending sessions
    SDB-->>P: pending session list
    P->>SDB: UPDATE session status=active
    SDB-->>P: OK

    C->>SDB: SELECT status (poll)
    SDB-->>C: status=active ✓

    Note over C,P: WebRTC Initialization Phase
    C->>C: getUserMedia() — camera + mic
    P->>P: getUserMedia() — camera + mic

    C->>SRT: subscribe(video-call-{sessionId})
    SRT-->>C: SUBSCRIBED
    P->>SRT: subscribe(video-call-{sessionId})
    SRT-->>P: SUBSCRIBED

    Note over C,P: NAT Traversal
    C->>STUN: STUN binding request
    STUN-->>C: public IP:port
    P->>STUN: STUN binding request
    STUN-->>P: public IP:port

    Note over C,P: Signaling Phase
    C->>C: createOffer() + setLocalDescription()
    C->>SRT: broadcast {type: offer, sdp}
    SRT->>P: relay offer
    P->>P: setRemoteDescription(offer)
    P->>P: createAnswer() + setLocalDescription()
    P->>SRT: broadcast {type: answer, sdp}
    SRT->>C: relay answer
    C->>C: setRemoteDescription(answer)

    Note over C,P: ICE Candidate Exchange
    C->>SRT: broadcast ICE candidates
    SRT->>P: relay ICE candidates
    P->>SRT: broadcast ICE candidates
    SRT->>C: relay ICE candidates

    Note over C,P: Video Call Active (P2P — SRTP encrypted)
    C<-->P: Audio/Video Stream (direct peer-to-peer)

    Note over C,P: Call Termination
    C->>SRT: broadcast {type: hang-up}
    SRT->>P: relay hang-up
    C->>C: cleanup() — close PeerConnection
    P->>P: cleanup() — close PeerConnection

    C->>SDB: UPDATE session status=completed
    C->>C: Show Rating Modal (1-5 stars)
    C->>SDB: UPDATE session rating=N
    C->>SDB: UPDATE psychiatrist avg rating

    Note over C,P: Both navigate to Home Screen

Fig 3.10 — State Diagram (Session Lifecycle)

stateDiagram-v2
    [*] --> Idle

    Idle --> Pending : Client requests session
    Pending --> Active : Psychiatrist accepts
    Pending --> Cancelled : Psychiatrist declines /
Client cancels / Timeout

    Active --> Connected : WebRTC P2P connection
established

    Connected --> Completed : Either party
ends call

    Completed --> Rated : Client submits
rating (1-5 stars)
    Completed --> Idle : Client skips rating

    Rated --> Idle : Return to home screen

    Cancelled --> Idle : Return to home screen

    state Idle {
        [*] --> Browsing
        Browsing --> SelectingDoctor
        SelectingDoctor --> PaymentProcessing
        PaymentProcessing --> [*]
    }

    state Active {
        [*] --> InitializingWebRTC
        InitializingWebRTC --> ExchangingSignals
        ExchangingSignals --> NegotiatingICE
        NegotiatingICE --> [*]
    }

    state Connected {
        [*] --> VideoCallActive
        VideoCallActive --> MutedAudio : Toggle Mute
        MutedAudio --> VideoCallActive : Toggle Unmute
        VideoCallActive --> CameraOff : Toggle Camera
        CameraOff --> VideoCallActive : Toggle Camera On
    }

Fig 4.1 — Physical Database Schema

erDiagram
    users {
        UUID id PK "Primary Key"
        VARCHAR firebase_uid UK "Unique - Firebase link"
        VARCHAR email "Nullable"
        VARCHAR display_name "Nullable"
        VARCHAR role "client / psychiatrist / admin"
        BOOLEAN is_anonymous "Default: false"
        VARCHAR alias "e.g. CalmOwl#7392"
        TIMESTAMPTZ created_at "Default: now()"
    }

    psychiatrists {
        UUID id PK "Primary Key"
        UUID user_id FK "FK to users.id"
        VARCHAR full_name "Required"
        VARCHAR license_number "Required, min 5 chars"
        TEXT license_image_url "Supabase Storage URL"
        VARCHAR specialization "Required"
        INTEGER experience "Years, 0-50"
        TEXT bio "Min 20 characters"
        INTEGER price "INR 99-499"
        BOOLEAN is_approved "Default: false"
        BOOLEAN is_online "Default: false"
        DECIMAL rating "Default: 5.0"
        INTEGER total_sessions "Default: 0"
        TIMESTAMPTZ created_at "Default: now()"
    }

    sessions {
        UUID id PK "Primary Key"
        UUID client_id FK "FK to users.id"
        UUID psychiatrist_id FK "FK to psychiatrists.id"
        VARCHAR status "pending/active/completed/cancelled"
        VARCHAR type "chat / video"
        INTEGER rating "1-5, Nullable"
        TIMESTAMPTZ started_at "Nullable"
        TIMESTAMPTZ ended_at "Nullable"
        TIMESTAMPTZ created_at "Default: now()"
    }

    messages {
        UUID id PK "Primary Key"
        UUID session_id FK "FK to sessions.id"
        UUID sender_id FK "FK to users.id"
        TEXT text "Message content"
        TIMESTAMPTZ created_at "Default: now()"
    }

    notifications {
        UUID id PK "Primary Key"
        UUID user_id FK "FK to users.id"
        VARCHAR title "Notification title"
        TEXT body "Notification body"
        JSONB data "Routing metadata"
        BOOLEAN read "Default: false"
        TIMESTAMPTZ created_at "Default: now()"
    }
